## 1. 逻辑回归

### 1.1 它解决什么问题

逻辑回归用于二分类。本项目用它估计“下一交易日上涨”的概率。

```text
输入：当前交易日的 15 个特征
输出：下一交易日上涨概率 p
分类：p >= 0.5 -> 1，否则 -> 0
```

### 1.2 为什么名字叫回归，却用于分类

模型先对特征做线性组合：

```text
z = w1*x1 + w2*x2 + ... + wn*xn + b
```

再通过 sigmoid 把任意实数压缩到 `0` 到 `1`：

```text
sigmoid(z) = 1 / (1 + exp(-z))
```

它拟合的是上涨概率或对数几率，而最终用途是分类。

### 1.3 权重、偏置与概率

- 权重 `w`：某个特征对结果的影响方向和程度。
- 偏置 `b`：整体基准倾向。
- 概率 `p`：模型认为属于上涨类别的程度。

逻辑回归还可以写为：

```text
log[p / (1-p)] = w*x + b
```

### 1.4 为什么适合作为基线

逻辑回归结构简单、训练快、易解释。如果复杂模型没有显著优于它，就说明复杂度未必带来更强泛化。

### 1.5 本项目中怎么使用

文件：`src/ml_models.py`

```python
LogisticRegression(max_iter=5000, random_state=42)
```

输入包含 OHLCV 和 9 项技术特征。数据按时间顺序切分为前 80% 训练、后 20% 测试。

### 1.6 结果怎么解释

| 指标 | 值 |
|---|---:|
| accuracy | 0.5886 |
| precision | 0.5886 |
| recall | 1.0000 |
| F1 | 0.7410 |

测试集上涨比例也是 `0.5886`。实际预测文件显示，逻辑回归把 `542` 个测试样本全部预测为上涨。

因此更准确的结论是：逻辑回归在当前设定下退化成了“总猜上涨”的基线。不能仅凭 accuracy 宣称模型有效。输入未标准化且尺度差异较大，也可能影响模型训练与解释。

### 1.7 老师可能怎么问

**问：为什么逻辑回归准确率最高，却不能直接说它最好？**

答：因为它的 recall 为 1，并且实际将全部测试样本预测为上涨。测试集上涨比例正好是 58.86%，所以准确率主要来自类别分布，不代表稳定识别了涨跌规律。

## 2. 随机森林

### 2.1 决策树是什么

决策树像一组逐层判断：

```text
如果 volatility_20 < 某阈值：
    再判断 return_1d
否则：
    再判断 close_ma20_ratio
```

树通过特征阈值把样本不断分组，直到叶子节点给出类别。

### 2.2 为什么随机森林是多棵树

单棵树容易依赖某些样本。随机森林训练多棵树，并对结果投票：

```text
prediction = majority_vote(tree_1, tree_2, ..., tree_n)
```

### 2.3 bagging 思想

Bagging 是 Bootstrap Aggregating：

1. 对训练数据有放回抽样。
2. 训练多个子模型。
3. 汇总多个子模型结果。

随机森林还会在节点划分时随机选择部分特征，从而增加树之间差异。

### 2.4 为什么能拟合非线性

决策树依靠多个阈值切分特征空间，不要求输入与输出之间是线性关系。因此它能描述更复杂的条件组合。

### 2.5 为什么仍可能泛化不好

- 金融短期信号本身弱。
- 训练期与测试期市场分布可能变化。
- 参数不一定适合当前样本。
- 技术特征高度相关。
- 模型可能学习训练期局部规则，却难以迁移到测试期。

### 2.6 本项目中怎么使用

文件：`src/ml_models.py`

```python
RandomForestClassifier(
    n_estimators=100,
    max_depth=10,
    random_state=42,
    n_jobs=-1,
)
```

### 2.7 结果怎么解释

| 指标 | 值 |
|---|---:|
| accuracy | 0.4188 |
| precision | 0.6429 |
| recall | 0.0282 |
| F1 | 0.0541 |

实际预测中，RF 只预测了 `14` 个上涨日，`528` 天预测为不上涨。它对上涨类别识别不足，当前泛化能力较弱。

### 2.8 老师可能怎么问

**问：随机森林更复杂，为什么反而更差？**

答：复杂度不等于泛化能力。当前特征主要来自历史行情，短期信号较弱，训练期和测试期还可能存在分布变化。随机森林在当前参数和样本下对上涨类别识别不足。

## 3. LSTM

### 3.1 RNN 是什么

普通前馈网络把每个样本独立处理。RNN 会把前一个时间步的隐藏状态传给下一个时间步：

```text
h(t) = f(x(t), h(t-1))
```

这使模型可以利用时间顺序。

### 3.2 普通 RNN 的问题

序列较长时，梯度可能消失或爆炸，较早信息难以稳定保留。LSTM 通过门控机制缓解这一问题。

### 3.3 LSTM 的门控机制

LSTM 的核心思想是：不是所有旧信息都保留，也不是所有新信息都写入。

| 组件 | 直觉 |
|---|---|
| 遗忘门 `f_t` | 旧记忆保留多少 |
| 输入门 `i_t` | 新信息写入多少 |
| 候选记忆 | 准备写入的新内容 |
| 细胞状态 `c_t` | 跨时间传递的长期记忆 |
| 输出门 `o_t` | 当前对外输出多少 |
| 隐藏状态 `h_t` | 当前时间步的输出表达 |

常见公式：

```text
f_t = sigmoid(W_f * [h_(t-1), x_t] + b_f)
i_t = sigmoid(W_i * [h_(t-1), x_t] + b_i)
c~_t = tanh(W_c * [h_(t-1), x_t] + b_c)
c_t = f_t * c_(t-1) + i_t * c~_t
o_t = sigmoid(W_o * [h_(t-1), x_t] + b_o)
h_t = o_t * tanh(c_t)
```

### 3.4 20 日窗口如何形成样本

项目使用 9 个技术特征和 `lookback=20`：

```text
X[0] = 第 1 至 20 个交易日的特征
y[0] = 第 21 个位置对应的涨跌标签
```

每向后移动一天，就形成一个新窗口。

```text
训练 X：(2144, 20, 9)
测试 X：(522, 20, 9)
```

### 3.5 本项目模型

文件：`src/lstm_model.py`

```text
单层 LSTM(input_size=9, hidden_size=32)
-> 取最后时间步
-> Linear(32, 1)
-> Sigmoid
```

训练参数：

| 参数 | 值 |
|---|---:|
| lookback | 20 |
| hidden size | 32 |
| batch size | 32 |
| epochs | 30 |
| learning rate | 0.001 |

### 3.6 标签修复为什么重要

当前正确代码：

```python
df["target_return_1d"] = df["return_1d"].shift(-1)
df = df.dropna(subset=["target_return_1d"])
df["target_direction"] = (df["target_return_1d"] > 0).astype(int)
```

末行没有下一交易日收益。若先比较再删除，`NaN > 0` 会变成 `False`，末行可能被误标为下跌。

### 3.7 结果怎么解释

| 指标 | 值 |
|---|---:|
| accuracy | 0.4540 |
| precision | 0.6173 |
| recall | 0.1645 |
| F1 | 0.2597 |
| AUC | 0.4916 |

AUC 接近 `0.5`，表示当前排序区分能力接近随机。LSTM 没有体现出比简单模型更强的短期预测能力。

### 3.8 老师可能怎么问

**问：LSTM 适合序列，为什么效果仍然不好？**

答：适合处理序列不等于一定预测准确。金融短期收益噪声较高，项目只使用基础技术特征，单层 LSTM 参数也较基础。结果主要说明当前输入下的可预测信号有限。

## 4. Q-learning

### 4.1 什么是强化学习

强化学习关注智能体与环境交互：

```text
观察状态 -> 选择动作 -> 获得奖励 -> 进入下一状态 -> 更新策略
```

目标不是只预测标签，而是学习长期累计奖励较高的行为。

### 4.2 Q-learning 是什么

Q-learning 学习一个 Q 表：

```text
Q(state, action) = 在状态 state 采取 action 的长期价值估计
```

### 4.3 本项目六要素

| 要素 | 本项目实现 |
|---|---|
| agent | 表格型 Q-learning 智能体 |
| environment | `StockTradingEnv` |
| state | `return_1d`、`close_ma20_ratio`、`volatility_20` 三变量离散化组合 |
| action | `0=空仓`、`1=持有` |
| reward | 下一期持仓收益减去变仓成本 |
| policy | 训练时 epsilon-greedy，评估时贪心 |

### 4.4 为什么离散化

表格型 Q-learning 需要有限状态。项目将三个连续变量各分成三档：

```text
3 * 3 * 3 = 27 个状态
```

如果直接把每个浮点值当状态，几乎每一天都是新状态，Q 表无法有效复用经验。

### 4.5 epsilon-greedy

```text
以 epsilon 概率：随机动作，探索
以 1-epsilon 概率：选择 Q 值最大动作，利用
```

本项目配置：

```text
epsilon_start = 0.30
epsilon_min   = 0.05
epsilon_decay = 0.995
episodes      = 50
```

由于训练只有 50 轮，实际最终 epsilon 为 `0.2335`，没有降到下限 `0.05`。

### 4.6 更新公式

```text
Q(s,a) <- Q(s,a) + alpha * [r + gamma * max Q(s',a') - Q(s,a)]
```

直觉：

1. 原本对动作价值有一个估计。
2. 执行动作后得到即时奖励。
3. 再参考下一状态的最佳未来价值。
4. 用学习率逐步修正旧估计。

项目参数：

| 参数 | 值 | 含义 |
|---|---:|---|
| `alpha` | 0.1 | 学习率 |
| `gamma` | 0.95 | 未来奖励折扣 |
| `episodes` | 50 | 重复训练轮数 |

### 4.7 为什么不是 DQN

表格型 Q-learning 直接保存有限状态的 Q 值。DQN 用神经网络近似 Q 函数，适合更复杂状态空间。项目选择表格型方法，是为了控制课程范围，并把状态、动作、奖励和更新公式讲清楚。

### 4.8 为什么没有使用 PPO

PPO 属于更复杂的策略优化算法。当前课程目标不是堆叠模型，而是完成可解释的 RL 入门闭环。项目中未实现 PPO，也不需要新增。

### 4.9 结果怎么解释

| 策略 | total return | Sharpe | max drawdown |
|---|---:|---:|---:|
| Random | -57.6447% | -0.5415 | -64.0596% |
| Buy & Hold | +227.1229% | 0.7055 | -34.1047% |
| Q-learning | +6.5649% | 0.1119 | -18.0356% |

Q-learning 没有超过买入持有。它只说明在当前简化环境中形成了一定行为规则。尤其要注意：RL 在同一段历史数据上训练和评价，没有严格样本外测试。

### 4.10 老师可能怎么问

**问：Q-learning 没有超过买入持有，为什么还值得做？**

答：课程设计重点是展示强化学习交易问题如何定义，包括环境、状态离散化、动作、奖励、探索和价值更新。结果弱于买入持有也是客观结论，说明简化状态和有限训练存在局限。

## 5. 指标解释

### 5.1 分类指标

先定义：

| 名称 | 含义 |
|---|---|
| TP | 实际上涨，预测上涨 |
| FP | 实际不上涨，预测上涨 |
| TN | 实际不上涨，预测不上涨 |
| FN | 实际上涨，预测不上涨 |

| 指标 | 公式 | 直觉 | 趋势 | 本项目怎么读 |
|---|---|---|---|---|
| Accuracy | `(TP+TN)/(TP+FP+TN+FN)` | 总体猜对比例 | 通常越大越好，但需结合类别分布 | LR 0.5886 主要来自全猜上涨 |
| Precision | `TP/(TP+FP)` | 预测上涨中有多少真的上涨 | 通常越大越好 | RF 0.6429，但预测上涨样本很少 |
| Recall | `TP/(TP+FN)` | 实际上涨中有多少被找到 | 通常越大越好 | LR 1.0 因为全部预测上涨；RF 0.0282 很低 |
| F1 | `2PR/(P+R)` | precision 与 recall 的综合平衡 | 通常越大越好 | RF 0.0541 说明识别上涨能力弱 |
| AUC | ROC 曲线下面积 | 随机抽一正一负样本，正样本排序更高的概率 | 越接近 1 越好；0.5 接近随机 | LSTM 0.4916，区分能力弱 |

### 5.2 收益与风险指标

| 指标 | 公式 | 直觉 | 趋势 | 本项目怎么读 |
|---|---|---|---|---|
| Annual return | `mean(daily_return)*252` | 年化后的平均收益估计 | 不能脱离风险单独看 | SPY 12.5362% |
| Annual volatility | `std(daily_return)*sqrt(252)` | 年化波动 | 风险角度通常越小越稳 | SPY 17.8595% |
| Sharpe ratio | `annual_return/annual_volatility` | 单位波动对应收益 | 一般越大越好 | SPY 0.7019；未扣无风险利率 |
| Max drawdown | `min((equity-running_max)/running_max)` | 历史最深回撤 | 越接近 0 越好 | SPY -34.1047% |
| Calmar ratio | `annual_return/abs(max_drawdown)` | 收益相对最大回撤 | 一般越大越好 | SPY 0.3676 |
| VaR 95% | 日收益率 5% 分位数 | 尾部损失门槛 | 风险角度越接近 0 越好 | SPY -1.6773% |
| CVaR 95% | `mean(return <= VaR)` | 最差尾部日的平均损失 | 风险角度越接近 0 越好 | SPY -2.7319% |

## 6. 一组必须会答的对比

| 问题 | 回答重点 |
|---|---|
| LR 和 RF 的差别 | LR 偏线性、易解释；RF 用多树拟合非线性，但复杂度不保证泛化 |
| 普通 ML 和 LSTM 的差别 | 普通 ML 一行特征对应一个样本；LSTM 输入连续 20 天序列 |
| 监督学习和 RL 的差别 | 监督学习拟合标签；RL 根据奖励学习动作价值 |
| Q-learning 和 DQN 的差别 | Q-learning 使用表格；DQN 用神经网络近似 Q 函数 |
| Accuracy 和收益的差别 | Accuracy 是分类命中率；收益还受仓位、时序、成本和风险影响 |
| 正收益和投资建议的差别 | 历史课程实验不等于未来盈利能力，不能作为投资建议 |
