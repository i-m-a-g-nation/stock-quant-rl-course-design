# 基于机器学习与强化学习的股票交易策略建模及量化风险评估研究

摘  要：本课程设计以标准普尔500指数ETF（SPY）为对象，完成了数据获取、清洗、特征工程、监督学习、LSTM、回测、风险评估和Q-learning交易实验的完整流程。为提高结论可靠性，实验补充了DummyClassifier基线、balanced_accuracy、ROC-AUC、混淆矩阵、Permutation Importance、多策略回测以及Q-learning训练集/测试集分离评估。结果显示，LogisticRegression的accuracy为0.5886，与DummyMostFrequent的0.5886一致，且二者均100%预测上涨，说明该模型未真正优于多数类基线；LSTM的ROC-AUC为0.4916，短期方向区分能力有限。样本外测试中，Q-learning总收益为-9.59%，未跑赢BuyAndHold，但最大回撤低于BuyAndHold。总体来看，短期方向预测难度较高，复杂模型不必然优于简单基线。本项目仅用于课程实验和方法验证，不构成实际投资建议。

关键词：股票交易；机器学习；LSTM；Q-learning；量化回测；风险评估

## 1. 课设背景

### 1.1 任务阐述

本课程设计的任务是基于公开金融时间序列数据，构建一个覆盖数据处理、特征构造、预测建模、交易回测和风险评估的实验型量化分析流程。项目名称为stock-quant-rl-course-design，研究题目为“基于机器学习与强化学习的股票交易策略建模及量化风险评估研究”。本项目不以构建真实可盈利交易系统为目标，而是通过SPY历史数据验证机器学习、深度学习和强化学习方法在金融数据挖掘场景中的基本流程、适用条件和局限性。

### 1.2 任务分析

股票价格序列具有噪声大、非平稳、受宏观环境和市场情绪影响显著等特点，直接预测未来价格容易受到价格尺度和趋势漂移影响。因此，本项目将建模任务转化为下一交易日涨跌方向的二分类问题，并用accuracy、balanced_accuracy、ROC-AUC、混淆矩阵和策略回测结果共同评估模型。该设定既符合课程中分类模型、时间序列特征工程和风险指标计算的教学目标，也能避免将模型输出误解为精确价格预测。

## 2. 课程设计方案

### 2.1 方案设计

项目采用分阶段实验设计：首先读取SPY历史行情数据并进行质量检查；其次构造收益率、移动平均、波动率、成交量和相对价格位置等特征；随后分别训练LogisticRegression、RandomForestClassifier和LSTM模型预测下一交易日涨跌方向；在此基础上将模型预测信号转化为仓位信号，开展简化回测并计算收益、波动、夏普比率、最大回撤、VaR和CVaR等指标；最后构建离散状态的Q-learning交易智能体，并在样本外测试集上与随机策略和买入持有策略进行对比。

### 2.2 数据集选择与准备

本项目选择SPY作为研究对象，主要原因包括：SPY跟踪美国大盘指数，流动性较高，历史数据相对完整；其价格走势能够较好代表宽基市场风险收益特征；相较于单只个股，SPY受个别公司事件影响较小，更适合作为课程实验中的方法验证对象。清洗后的特征数据包含2707行、23列，时间范围为2015-03-30至2025-12-31，缺失值总数为0。

### 2.4 模型设计方案

监督学习部分采用LogisticRegression和RandomForestClassifier。LogisticRegression通过线性决策边界估计上涨概率，具有结构简单、可解释性较强的特点；RandomForestClassifier通过多棵决策树集成捕捉非线性关系，但在金融时间序列样本中可能出现泛化能力不足或类别预测偏置。为避免只看accuracy造成误判，当前版本加入DummyMostFrequent和DummyStratified作为基线模型，并补充balanced_accuracy、ROC-AUC和混淆矩阵。

深度学习部分采用LSTM模型处理固定长度时间窗口序列。LSTM通过门控结构缓解普通循环神经网络的长期依赖问题，理论上可以捕捉时间序列中的局部动态模式。本项目将其作为对传统机器学习模型的补充实验，不将其结果解释为稳定可盈利信号。

强化学习部分采用Tabular Q-learning。智能体根据离散化后的市场状态选择空仓或持有动作，并以收益扣除交易成本后的结果作为奖励。Q-learning通过贝尔曼更新学习状态-动作价值表，其价值在于验证交易环境、状态设计、奖励函数和样本外评估流程，而不是证明智能体能够在真实市场持续盈利。

### 2.5 回测与风险评估方案

回测部分将模型输出的上涨/下跌信号转换为持仓信号，并对信号进行shift(1)处理，避免使用当日尚不可得的信息。策略绩效从累计收益、年化收益、年化波动率、夏普比率、最大回撤、Calmar比率、胜率和交易次数等角度评价。风险评估部分重点解释最大回撤、VaR和CVaR：最大回撤反映净值从阶段高点回落的最大幅度，VaR衡量给定置信水平下的潜在损失阈值，CVaR进一步刻画超过VaR阈值时的平均尾部损失。

### 3.2 数据获取与预处理

数据处理流程包括读取原始行情、统一日期索引、检查缺失值和异常值、保存清洗结果以及生成价格走势、收益率分布和成交量图。当前数据质量报告显示，进入特征工程阶段的数据不存在缺失值，能够满足后续建模需求。

### 3.3 特征工程实现

特征工程代码位于src/features.py和run_stage2.py。实现过程按时间顺序计算收益率、移动平均、滚动波动率和成交量均线，并新增相对化特征列集合。为避免未来信息泄露，所有滚动指标均使用当前日及历史窗口数据构造，预测标签则使用下一交易日收益方向生成。

### 3.4 机器学习模型实现

机器学习模型实现位于src/ml_models.py和run_stage3.py。当前版本在原有逻辑回归和随机森林基础上新增DummyClassifier基线，并输出增强版指标表、混淆矩阵、测试集预测结果和Permutation Importance。模型训练和测试按时间顺序划分，避免随机划分破坏金融时间序列结构。

### 3.5 LSTM模型实现

LSTM实现位于src/lstm_model.py和run_stage5.py。模型将连续若干交易日特征作为输入序列，并预测下一交易日涨跌方向。项目已修复末行NaN标签问题，避免最后一行因不存在下一交易日收益而被错误纳入训练或评估。

### 3.6 回测与风险指标实现

回测实现位于src/backtest.py和run_stage4.py，风险指标计算位于src/risk_metrics.py。当前版本同时回测BuyAndHold、AlwaysLong、LogisticRegression、RandomForestClassifier、DummyMostFrequent和DummyStratified六类策略，并生成权益曲线、回撤曲线和策略指标表。

### 3.7 强化学习环境与Q-learning实现

强化学习实现位于src/trading_env.py、src/rl_agent.py和run_stage6.py。当前版本将数据按时间顺序划分为训练集和测试集，在训练集上使用epsilon-greedy策略学习Q-table，在测试集上采用epsilon=0的贪心策略评估，并与RandomPolicy和BuyAndHold进行对比。

### 4.6 综合分析

综合监督学习、LSTM、回测和强化学习结果可以发现，SPY短期涨跌方向预测具有较高难度，复杂模型不必然优于简单基线。LogisticRegression的accuracy虽然达到0.5886，但与多数类基线相同，说明其主要反映测试区间类别分布；RandomForestClassifier在降低风险暴露方面有一定特征，但预测上涨能力不足；LSTM在当前样本和参数设置下未体现明显优势；Q-learning完整验证了强化学习交易流程，但样本外收益为负，未跑赢买入持有。

回测结果还说明，在整体上涨的测试区间中，买入持有策略可能显著优于复杂主动交易策略。与此同时，SPY最大回撤达到-34.10%，说明高收益背后存在明显下行风险。因此，对金融模型的评价不能只看收益率，还需要结合回撤、波动率、交易成本、样本外表现和基线策略进行综合判断。

### 4.7 结论与展望

本课程设计完成了一个面向金融数据挖掘课程的端到端实验流程。实验结论表明：第一，SPY涨跌方向预测并不容易，监督学习模型在当前特征集上接近随机水平；第二，LSTM深度学习模型并未显著提升短期方向预测能力；第三，买入持有策略在测试区间内表现强于主动交易策略，但也伴随较大回撤；第四，Q-learning能够形成完整的状态、动作、奖励和策略评估流程，但在样本外测试中未获得优于买入持有的收益。

后续可从滚动训练、更多外部特征、更细粒度状态表示、多级仓位动作空间和更真实的交易约束等方向改进。但这些改进仍需严格样本外验证，不能直接等同于真实投资能力。本项目仅用于课程实验和方法验证，不构成任何实际投资建议。
